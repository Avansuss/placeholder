In GitHub projects werk je niet met work items, maar issues. Het is in essentie het zelfde. In plaats van dat je Epics/User stories/Sub items hebt, heb je issues en sub-issues. Om het wat overzichtelijker te maken zullen er **labels** gebruikt worden.

# User stories / Issues
Issues zullen gereserveerd zijn voor user stories.
In de project pagina, onder de “Backlog” tabblad(en dan ook weer de “Backlog“ status row), klik op de plus en begin het typen van de User story. Als je klaar bent kun je of op enter of “Create new issue” klikken. Schrijf een passende beschrijving en selecteer het label “User story”. Hierna kan je er tot slot een estimate aan houden en assignee’s als dat van toepassing is.

# Work items / Sub-issues
Sub-issues zijn de work items die aan een user story zullen hangen.
Klik op een Issue en zoek voor de knop “Create sub-issue”. Schrijf een titel en beschrijving. De titel hoeft niet als een user story geschreven te worden.
Tot slot selecteer je de label “Work item”.

>[!tip]
> Als je “Create more sub-issues” aanvinkt kan je makkelijker meer work items voor een issue aanmaken.
